Project Overview

This project involves software using a linear regression model and its related technical documentation, developed in 3 months. We look at new approaches as we move forward. We implement sprints for project management using Agile methodology to check in weekly. Developers and technical writers arrange regular check-ins to review progress and issues together.

This document acts as a primary reference for all internal stakeholders. We update it regularly as the project progresses.

The platforms we use for this project include:

GitHub

To run code and manage the project repository, technical writers interact with the software by updating the code weekly by asking the developer team. We ensure all teams stay on the same page with the developers’ progress by documenting chunk information in each sprint.

Taiga

To manage tasks, developers and technical writers create minor tasks underneath each main task to work on achievable milestones in each sprint.

Zoom

To do a sprint retrospective weekly. We set a recurring schedule to check progress and improve communication across interdisciplinary teams.

Teams

To conduct SME interviews when technical writers have some questions about the software and want to contact the developer for better explanations and answers.

Whatsapp

To communicate informally and update work progress between teams.

Linear Regression (concept + application to our specific project)

Linear regression is a simple yet practical statistical approach used in AI and data analysis. It predicts a dependent value using one or more independent variables and fits a linear equation to observed data.

Consider a graph with several data points distributed over it. Linear regression tries to create a straight line that best matches these points. The line may then be used to predict future data points. While it is an early kind of artificial intelligence, it is effective for understanding variable connections and creating basic projections.

Python structure and source

For the development of the project’s code, we are trying to follow a programing architecture called MVC which consists in separating the data from the graphical interface and making them communicate which each other to organize the interface.

We are also trying to separate the code into different modules. For doing that we separate the interface in different parts for example the table with the data will be a module, the 2 columns selector other, the handle missing data other… and in each module we apply the architecture described before.

For being able to control what are we showing in the screen, we created a main controller file called Interface.py where we program what will be on screen on each moment.

In GitHub we are working developing the code in a branch apart from the main branch and once we finish the sprint, we merge these 2 branches to upload the code to the main branch. For the commit (upload something to GitHub) we write a message anytime we upload something with the number of the user’s story of the task and try to explain what we did.

Code errors, problems, and their solutions

While developing the code we didn’t have many technical errors apart from libraries compatibility and things like that.

Otherwise, we had some troubles with the code when we had to readapt it to fit the MVC architecture and when we divided it in modules because separating code implies to rewrite parts of the code to make the different files work together, but with patient and working together we were able to fix the problems.

Maybe the biggest problem that we have had is adding a loading bar because we had so many errors and we couldn’t implement it but now we are making progress with by using new programming techniques.
